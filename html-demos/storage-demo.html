Web Storage 使用演示

📖 基础知识

Web Storage 提供两种机制：
  • sessionStorage：会话级存储，页面关闭时数据丢失
  • localStorage：持久化存储，除非手动清除，否则数据永久保存
特性 sessionStorage localStorage
生命周期 页面会话结束（关闭标签/浏览器） 永久存储，除非手动清除
共享范围 仅当前页面/标签 同源的所有页面/标签
存储大小 约 5MB 约 5MB

⚠️ 数据会被强制转为字符串的情况

重要：localStorage 和 sessionStorage 只能存储字符串类型。 调用 setItem(key, value) 时，所有非字符串值都会被自动调用 toString() 方法转换。

🔴 属性会丢失的情况

属性丢失场景：
  • 存储对象时，如果对象包含 function、Symbol、undefined 类型的值，这些属性会被 JSON.stringify() 忽略
  • 存储 Date 对象会变成字符串，无法恢复为 Date 实例
  • 存储 RegExp、Map、Set 等特殊对象会变成空对象 {}
